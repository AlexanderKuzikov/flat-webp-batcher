# flat-webp-batcher

Node.js-утилита для пакетной обработки изображений из папки `IN` и всех вложенных подпапок.

Утилита:
- рекурсивно обходит входную директорию;
- выбирает только те папки, в которых есть не меньше `minImagesInFolder` изображений;
- уменьшает изображения по ширине до `maxWidth` пикселей без upscale;
- сохраняет пропорции по высоте;
- конвертирует результат в WebP с качеством `webpQuality`;
- складывает все результаты в одну общую папку `OUT`;
- не сохраняет структуру исходных папок;
- при коллизиях имен автоматически добавляет суффиксы `__2`, `__3` и так далее;
- умеет работать с параллельной обработкой.

## Требования

- Node.js 18+
- npm

## Установка

```bash
npm install
```

## Запуск

```bash
npm start
```

или напрямую:

```bash
node src/index.js
```

## Конфигурация

Файл `config.json`:

```json
{
  "inputDir": "IN",
  "outputDir": "OUT",
  "maxWidth": 1024,
  "webpQuality": 75,
  "minImagesInFolder": 6,
  "concurrency": 0,
  "supportedExtensions": [
    ".jpg",
    ".jpeg",
    ".png",
    ".webp",
    ".tif",
    ".tiff",
    ".bmp",
    ".gif",
    ".avif"
  ]
}
```

### Параметры

- `inputDir` — входная папка. Можно указать относительный или абсолютный путь.
- `outputDir` — выходная папка. Можно указать относительный или абсолютный путь.
- `maxWidth` — максимальная ширина результирующего изображения.
- `webpQuality` — качество WebP от 1 до 100.
- `minImagesInFolder` — минимальное число изображений в папке, чтобы она попала в обработку.
- `concurrency` — число одновременно обрабатываемых файлов. Если `0`, утилита сама выбирает значение по числу доступных CPU.
- `supportedExtensions` — список расширений, которые считаются изображениями.

## Пути и слэши

Поддерживаются:
- относительные пути, например `IN` и `OUT`;
- абсолютные пути;
- пути на другом диске;
- прямые и обратные слэши.

Примеры для Windows:

```json
{
  "inputDir": "D:/photo/source",
  "outputDir": "E:\\photo\\out"
}
```

Примеры для Linux/macOS:

```json
{
  "inputDir": "/mnt/data/source",
  "outputDir": "/mnt/data/out"
}
```

## Логика отбора папок

Если в папке меньше `minImagesInFolder` изображений, все файлы из этой папки пропускаются.

Пример:
- `IN/a` содержит 3 изображения — папка пропускается;
- `IN/b` содержит 6 изображений — все изображения из `IN/b` попадают в обработку;
- `IN/c/nested` содержит 8 изображений — папка `nested` попадает в обработку независимо от соседних папок.

## Коллизии имен

Результаты сохраняются в одну плоскую папку без структуры подпапок.

Если в разных папках встречаются одинаковые имена файлов, утилита не перезаписывает результат, а добавляет суффикс:

- `photo.jpg` -> `photo.webp`
- второй `photo.png` -> `photo__2.webp`
- третий `photo.tif` -> `photo__3.webp`

## Поведение

- EXIF orientation учитывается.
- Увеличение маленьких изображений не выполняется.
- Если `OUT` находится внутри `IN`, папка `OUT` исключается из сканирования.
- Если `outputDir` не существует, она будет создана автоматически.
- Если `inputDir` не существует, утилита завершится с ошибкой.
